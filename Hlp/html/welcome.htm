Feature high-lights:

  • The only image viewer you need for viewing & browsing images.
  • Clean with no unnecessary menus and toolbars wasting desktop space.
  • Uncompromising image quality.
  • Small and very fast, highly tuned code.
  • Reads all common image file formats.
  • Image index feature with very fast thumbnail-icon creation.
  • Rotate, mirror, zoom, pan, adjust contrast, brightness, sharpness, hue, saturation, gamma, ...
  • Adjustments can be saved non-destructively (and reapplied next time you open it).
  • There's an image index feature with very fast thumbnail-icon creation.
  • It does not attempt to be a half-baked editor, format converter, or media player.
  • It does include a simple slide show scripting language.
  • Translated to several languages.
  • Open Source.

Features for the tech-nerds:

  • Deep color & HDR support: Specialized code paths for grayscale and color data, with 8- or 16-bit integer, or 32-bit floating point / color component - maintaining full precision for all image processing operations.
  • High quality dithering: Floyd-Steinberg dithering is used for superior perceived image quality when displaying high bit-depth images.
  • Asynchronous file I/O: Images are decompressed at the same time as data is being read - i.e. it doesn't wait for file reads to finish before starting to decode.
  • Speculative decoding & caching of images: While you view a file, the next file in the folder is invisibly being decoded, so that it can be brought up very quickly. And when you move to a new image, the old one is remembered (if memory allows), so that there's little delay if you toggle back and forth between images.
  • Data parallelism exploitation: Many heavy computational tasks are considerably sped up by using SEE, SSE2, SSE3 and SSE4 instructions. These "SIMD instructions" work on 4, 8 or even 16 numbers in parallel.
  • Task parallelism exploitation: Heavy computational work is considerably sped up by dividing big jobs into smaller tasks that are distributed on multiple CPU cores (if available). Work stealing and dynamic load balancing techniques ensure that all cores are busy.
  • Hand tuned code optimizations: Critical sections are optimized by hand.
  • Code branching: Common cases are handled using specialized code that is faster than the generic routines.
  • High quality resampling: All resampling (image resizing) is done using Catmull-Rom interpolation.
  • Text meta data handling: Text info tags can be read from several formats - Adobe XMP, IPTC IMM, EXIF, PNG, TIFF tags, PSD, GIF and JPEG comments.
  • Native code: x64 (64-bit) and x86 (32-bit) versions are available.

In the following section you will be given basic instructions on how to use the rather minimalistic user interface.

Quick Start

When you start the program (e.g. by using Start → All Programs → Image Eye), the first thing you see is an Open image file dialog. Here you browse for the image file that you want to view. When a file is selected, you'll see both a small preview of it and a bit of file information. A lot of different File formats are supported. The Index button brings up a special window that shows small thumbnail icons of all the images in the current folder. To see the image in full size, click View.

A special feature of Image Eye is the image window is free from scroll bars, tool bars, and menus, which otherwise would occupy precious space and distract from the image. Even the window title bar and the borders are hidden if you move the mouse cursor outside of the image (NB; this feature can be disabled if you don't like it).

If the image is larger than the window, the mouse cursor will change shape when it's on top of the window, indicating in which directions it can be scrolled. To scroll it, you press down the left mouse button and drag the image (as an alternative you can scroll with the number-keys on the numeric keyboard). If you have a mouse with a mouse wheel, you can use it too to scroll vertically.

When the cursor is a normal pointer you can move the whole window easily by dragging it with the left mouse button. If it is showing a scroll-cursor, hold down the Alt-key to get the pointer instead.

If you double-click with the left button on the image (or alternatively hit Alt-Enter on the keyboard), the window is maximized to full-screen mode (or restored to a window if it already was maximized).

Now you may ask: Where's the program menu? Click with the right mouse button anywhere in the window and it will pop-up! The complete set of menu commands are described in the Viewer commands and Index commands sections. There you can also learn about keyboard short-cuts.

E.g. try right click → View → Adjust image... where you can change such things as contrast, sharpness, color saturation, or do gamma correction. Note that you can also save these adjustments so that they will be remembered and automatically re-applied the next time that you open the image file (this does not modify the original image file in any way whatsoever - the adjustments are stored in a separate hidden '.iea' file).

There are many ways that you can open an image file:

  1. Run Image Eye.exe, select one ore more files in the dialog box that pops up and click View.

  2. Open an index window, then click on one of the thumb-nail icons. You can bring up an index window by either clicking the Index button in the Open image file dialog, or by right clicking on a folder in any part of the Windows shell (e.g. the Windows Explorer) and selecting View as Image Eye index.

  3. If Image Eye has been associated with a file type; then just open the file from the Windows Explorer.

  4. In a running Image Eye window, select Spawn new (new window) or Load new (replaces current) from the menu.

  5. Drag an image file from the Windows Explorer to an already open Image Eye window.

  6. In an open image window, press PgDn / PgUp (or Space / Back-Space, or mouse browse buttons), to display the Next / Previous image.

  7. Hold down the left mouse button and scroll with the mouse wheel to browse among the images in the folder.

A feature that deserves a special explanation is freeze mode. Using View → Freeze window, you can lock the image window into its current position on the desktop (with no menu bars or anything) and the only thing you can do with it then is right click on it to bring up a menu with a few select commands (e.g. select Freeze window again to unfreeze it). If you shut down Windows without manually closing a frozen window, it will automatically load and show in same position the next time you boot up. This can be useful for e.g. sticking photographs of near and dear ones onto your desktop.


Portable version

If you run Image Eye.exe on a computer where it has not been installed through its Setup.exe, then settings are not remembered - and nothing will be stored in the registry et c, thus leaving an entirely 'untouched' system. This is useful e.g. if you want to run it from a USB-key on a borrowed computer.
